Topic 1: Random Number Generators

Why This Matters

Simulation depends on randomness. RNGs are the engine behind every Monte Carlo method. If your RNG is bad, your results are worthless.

Exam Relevance

LCG cycle analysis and LFSR bit generation appear in nearly every exam. You must be able to trace cycles by hand quickly.

Prerequisite Recap

Modular arithmetic is the core tool here. Think of it like a clock: after 12 comes 1 again. The modulus $m$ sets the clock size. The sequence $Z_0, Z_1, Z_2, \ldots$ eventually repeats because there are only $m$ possible values.

Background Knowledge: Modular Arithmetic & Sequences
Modular Arithmetic (Clock Analogy)

The operation $a \bmod m$ returns the remainder when $a$ is divided by $m$. Think of a clock with $m$ positions numbered $0, 1, \ldots, m{-}1$. After reaching $m{-}1$, the next step wraps back to $0$.

$17 \bmod 5 = 2$ because $17 = 3 \times 5 + 2$. On a "clock-5," position 17 is the same as position 2.
The Mod Operator in LCGs

In a Linear Congruential Generator, the recurrence $Z_{n+1} = (aZ_n + c) \bmod m$ means: multiply, add, then take the remainder. The modulus $m$ bounds all outputs to $\{0, 1, \ldots, m{-}1\}$.

With $a=3, c=2, m=5, Z_0=1$: $Z_1 = (3 \cdot 1 + 2) \bmod 5 = 0$, $Z_2 = (3 \cdot 0 + 2) \bmod 5 = 2$, $Z_3 = (3 \cdot 2 + 2) \bmod 5 = 3$, etc.
Sequences & Periodicity

Since there are only $m$ possible values, the sequence must eventually revisit a value — and from that point, it repeats. The period is the length of this repeating cycle. The tail (if any) is the non-repeating prefix before the cycle begins.

Sequence: $1, 3, 0, 2, 3, 0, 2, 3, \ldots$ — the tail is $\{1\}$ (length 1), the cycle is $\{3, 0, 2\}$ (period 3).

Write Down the Recurrence
State the LCG formula: $Z_{n+1} = (aZ_n + c) \bmod m$ with the given parameters $a$, $c$, $m$, and seed $Z_0$.
$Z_{n+1} = (aZ_n + c) \bmod m$
Compute $Z_1$
Substitute $Z_0$ into the formula. First compute $aZ_0 + c$, then take $\bmod\; m$. Write down the result.
Show the intermediate product before taking mod — examiners want to see your working.
Continue the Sequence
Feed each new $Z_n$ back into the formula to get $Z_{n+1}$. Build a table: $n \mid Z_n \mid aZ_n + c \mid Z_{n+1}$.
Detect the Repeat
Stop when you see a value that has appeared before. Circle the repeated value — the cycle runs from its first occurrence to just before the repeat.
Don't confuse the tail with the cycle! The period only counts the repeating part.
Count the Period
The period = number of values in the cycle. Maximum possible period is $m$ (full period). Report the cycle explicitly, e.g., "$\{3, 0, 2\}$, period = 3."
Full period ($= m$) requires careful choice of $a$, $c$, $m$. The Hull-Dobell theorem gives conditions.

Theory

Linear Congruential Generator (LCG)

$$Z_{n+1} = (aZ_n + c) \bmod m$$
  • $Z_n$: current number, $a$: multiplier, $c$: increment, $m$: modulus
  • $U_n = Z_n / m$ gives pseudo-random $U(0,1)$ numbers
  • Period $\leq m$; full period $m$ achievable via Hull-Dobell theorem

Linear Feedback Shift Register (LFSR)

$$b_i = (b_{i-r} + b_{i-q}) \bmod 2$$
  • Uses shift register with XOR feedback at tap positions $q$ and $r$
  • 8-bit integers: convert seed to binary, shift, XOR at positions $q$ and $r$
  • Non-overlapping bit strings for integer generation: take next 8 fresh bits

Common Mistakes

  • Forgetting that LCG period $\leq m$
  • Confusing "cycle" with "period" — the seed might not be on the cycle (it may be a tail leading into the cycle)
  • LFSR: forgetting non-overlapping vs overlapping bit interpretation

Topic 2: Testing RNGs & Input Distributions

Why This Matters

Having an RNG is not enough — you must verify it produces truly random-looking output. These tests detect patterns that would invalidate your simulation.

Exam Relevance

Runs test, Poker test, Chi-squared, and K-S test calculations appear regularly. Know when to use each and the correct degrees of freedom / adjusted statistics.

Prerequisite Recap

Hypothesis testing refresher: $H_0$ = "RNG is good". We compute a test statistic and compare to a critical value. If the statistic exceeds the critical value, we reject $H_0$. Think of it as "innocent until proven guilty" — we need strong evidence to reject.

Background Knowledge: Hypothesis Testing Framework
Null & Alternative Hypotheses

$H_0$ (null) is the "status quo" claim we assume true. $H_1$ (alternative) is what we're trying to find evidence for. We never "prove" $H_0$ — we either reject it or fail to reject it.

Testing a die: $H_0$: the die is fair (all faces equally likely). $H_1$: the die is not fair. We collect data and look for evidence against $H_0$.
Type I & Type II Errors

Type I error ($\alpha$): Rejecting $H_0$ when it's actually true (false alarm). Type II error ($\beta$): Failing to reject $H_0$ when it's actually false (missed detection). We control $\alpha$ by choosing a significance level (typically 0.05).

$\alpha = 0.05$ means we accept a 5% chance of wrongly rejecting a good RNG. Reducing $\alpha$ makes us more conservative but increases $\beta$.
Chi-Squared Distribution

The $\chi^2$ distribution with $k$ degrees of freedom arises when summing $k$ squared standard normals. It measures "total squared deviation" and is always non-negative. Larger values = more deviation from expected.

$\chi^2 = \sum_{i=1}^{k} \frac{(O_i - E_i)^2}{E_i}$ — each term measures how far an observed count $O_i$ is from its expected count $E_i$, scaled by $E_i$.
Critical Values & p-Values

The critical value $\chi^2_{\alpha,k}$ is the threshold: reject $H_0$ if the test statistic exceeds it. The p-value is the probability of seeing a result this extreme or more, assuming $H_0$ is true. Reject if $p < \alpha$.

At $\alpha = 0.05$ with $k = 4$ d.f., the critical value is $\chi^2_{0.05,4} = 9.488$. If your statistic is 11.2, reject $H_0$ (11.2 > 9.488).
Background Knowledge: Normal CDF & Standardisation
Standardisation

To convert any normal $X \sim N(\mu, \sigma^2)$ to the standard normal $Z \sim N(0,1)$, use $z = \frac{x - \mu}{\sigma}$. This lets you use a single table for all normal distributions.

If $X \sim N(2, 9)$ (mean 2, variance 9, so $\sigma = 3$), then $P(X \leq 5) = P\!\left(Z \leq \frac{5-2}{3}\right) = \Phi(1.00) = 0.8413$.
Reading the Normal Table

The standard normal table gives $\Phi(z) = P(Z \leq z)$. For negative $z$, use $\Phi(-z) = 1 - \Phi(z)$. For bin probabilities: $P(a < X < b) = \Phi\!\left(\frac{b - \mu}{\sigma}\right) - \Phi\!\left(\frac{a - \mu}{\sigma}\right)$.

$P(-1 < X < 5)$ with $X \sim N(2,9)$: $= \Phi\!\left(\frac{5-2}{3}\right) - \Phi\!\left(\frac{-1-2}{3}\right) = \Phi(1.00) - \Phi(-1.00) = 0.8413 - 0.1587 = 0.6827$.
Background Knowledge: Bin Merging Rule for Chi-Squared Tests
When to Merge Bins

The chi-squared approximation requires that each expected count $E_i \geq 5$. If any bin has $E_i < 5$, merge it with an adjacent bin until the combined expected count reaches 5. After merging, recount $k$ and recompute degrees of freedom $\text{df} = k' - 1 - p$ where $p$ is the number of estimated parameters.

Bins with $E = \{3, 12, 25, 18, 2\}$: merge first two and last two to get $E' = \{15, 25, 20\}$. Now $k' = 3$ and $\text{df} = 3 - 1 = 2$ (if no estimated parameters).

Visual Guide: How to Perform a Chi-Squared GoF Test

Bin the Data
Divide the observed data into $k$ categories (bins). Count the number of observations $O_i$ in each bin.
Ensure each bin has expected count $E_i \geq 5$. Merge small bins if necessary.
Compute Expected Counts
Under $H_0$, calculate the expected count for each bin: $E_i = n \cdot p_i$, where $n$ is the total sample size and $p_i$ is the theoretical probability for bin $i$.
$E_i = n \cdot p_i$
Calculate the Test Statistic
Compute the chi-squared statistic by summing the squared deviations, each divided by the expected count.
$\chi^2_0 = \sum_{i=1}^{k} \frac{(O_i - E_i)^2}{E_i}$
Find the Critical Value
Look up $\chi^2_{\alpha,\,k-1}$ from the chi-squared table. Degrees of freedom $= k - 1$ (subtract 1 more for each estimated parameter).
d.f. = $k - 1 - (\text{number of estimated parameters})$. Forgetting to subtract estimated params is a common error!
Decide: Reject or Fail to Reject
If $\chi^2_0 > \chi^2_{\alpha,\,k-1}$, reject $H_0$ — the data does not fit the hypothesised distribution. Otherwise, fail to reject (data is consistent with $H_0$).
"Fail to reject" ≠ "accept." We simply lack sufficient evidence against $H_0$.

Runs Test Steps:

Step 1: Count $a$ = values above 0.5, $b$ = values below 0.5, $n = a + b$.
Step 2: Count runs $r$ = number of consecutive groups above/below 0.5.

Step 1: Take $5n$ observations, convert to digits $y_k = \lfloor 10u_k \rfloor$.
Step 2: Group into $n$ sets of 5 digits each.
Step 3: Classify each group: all-different, 1-pair, 2-pair, 3-of-a-kind, full-house, 4-of-a-kind, 5-of-a-kind.
Step 4: Compute expected counts $np_j$ using the probability table.
Step 5: $\chi^2 = \sum \frac{(N_j - np_j)^2}{np_j}$, df = $k-1$. Compare to $\chi^2$ critical value.

Chi-Squared Test Steps:

Step 1: Bin the data into $k$ categories. Count observed $N_j$.
Step 2: Compute expected counts $np_j$ under $H_0$.
Step 3: $\chi^2 = \sum_{j=1}^k \frac{(N_j - np_j)^2}{np_j}$
Step 4: df = $k - 1$ (no estimated params) or $k - 1 - s$ ($s$ estimated params).
Step 5: Reject if $\chi^2 > \chi^2_{\text{df}, 1-\alpha}$.

K-S Test Steps:

Step 1: Sort data: $X_{(1)} \leq X_{(2)} \leq \ldots \leq X_{(n)}$.
Step 2: Compute $F(X_{(i)})$ for each observation.
Step 3: $D^+ = \max_i\!\left(\frac{i}{n} - F(X_{(i)})\right)$, $D^- = \max_i\!\left(F(X_{(i)}) - \frac{i-1}{n}\right)$
Step 4: $D = \max(D^+, D^-)$.
Step 5: Choose adjusted formula based on parameter estimation case. Compare to critical value.

Common Mistakes

  • Using wrong degrees of freedom for chi-squared ($k-1$, not $k$)
  • Forgetting to sort data for K-S test
  • Not using the correct adjusted test statistic for K-S (depends on whether parameters are known or estimated)
  • Runs test: forgetting the 0.5 continuity correction in the mean formula

Topic 3: Poisson Processes

Why This Matters

Queuing, network traffic, radioactive decay — Poisson processes model random arrivals everywhere. They are the bridge between probability theory and simulation practice.

Exam Relevance

Memoryless property, merging/splitting, and the hitchhiker's paradox are exam favourites. Practice conditional probability with exponentials.

Prerequisite Recap

An exponential distribution is the continuous analogue of "waiting for the next event." If events arrive at rate $\lambda$ per hour, the time between events is $\text{Exp}(\lambda)$ with mean $1/\lambda$. It is the only continuous memoryless distribution.

Background Knowledge: Exponential Distribution & Memorylessness
Exponential Distribution

The $\text{Exp}(\lambda)$ distribution models the time between events in a Poisson process. PDF: $f(x) = \lambda e^{-\lambda x}$ for $x \geq 0$. CDF: $F(x) = 1 - e^{-\lambda x}$. Mean: $1/\lambda$. Variance: $1/\lambda^2$.

If buses arrive at rate $\lambda = 3$/hour, the time between buses is $\text{Exp}(3)$ with mean $1/3$ hour = 20 minutes.
Memoryless Property

The exponential distribution satisfies $P(X > s + t \mid X > s) = P(X > t)$. "Given you've already waited $s$ minutes, the remaining wait time has the same distribution as if you just started." No other continuous distribution has this property.

If you've waited 10 min for a bus ($\lambda = 3$/hr), the probability of waiting another 5 min is the same as the probability of waiting 5 min from scratch: $e^{-3 \cdot 5/60}$.
Event Counting vs Inter-Arrival Times

Two views of the same process: (1) Count arrivals in $[0, t]$ → $N(t) \sim \text{Poisson}(\lambda t)$; (2) Measure inter-arrival times $T_i$ → $T_i \sim \text{Exp}(\lambda)$, i.i.d. These are equivalent descriptions.

$\lambda = 5$/hr over 2 hours: $N(2) \sim \text{Poisson}(10)$, and each gap between arrivals is $\text{Exp}(5)$.

Visual Guide: How to Simulate & Merge Poisson Processes

Generate Exponential Inter-Arrival Times
For each process with rate $\lambda$, generate inter-arrival times using the inverse transform: $T_i = -\frac{1}{\lambda} \ln(1 - U_i)$ where $U_i \sim U(0,1)$.
$T_i = -\frac{1}{\lambda} \ln(1 - U_i)$
Accumulate Arrival Times
Convert inter-arrival times to absolute arrival times by cumulative summing: $A_n = \sum_{i=1}^{n} T_i$. This gives the actual clock time of each event.
For Merging: Interleave Sorted Times
To merge two Poisson processes ($\lambda_1$ and $\lambda_2$), combine all arrival times from both processes and sort them in increasing order. The merged sequence is itself a Poisson process.
Identify Source of Each Event
Each event in the merged process came from process 1 with probability $\frac{\lambda_1}{\lambda_1 + \lambda_2}$ and process 2 with probability $\frac{\lambda_2}{\lambda_1 + \lambda_2}$, independently.
This is the splitting property in reverse — merging is the converse of thinning.
Compute the Merged Rate
The merged process has rate $\lambda_1 + \lambda_2$. Verify: the combined inter-arrival times should have mean $\frac{1}{\lambda_1 + \lambda_2}$.
$\lambda_{\text{merged}} = \lambda_1 + \lambda_2$

Theory

Poisson Process Properties

  1. Independent increments
  2. Stationarity: $N(t)$ depends only on interval length
  3. No simultaneous events

Poisson PMF

FS: Distributions $$P[N(t) = k] = \frac{(\lambda t)^k}{k!} e^{-\lambda t}, \quad k = 0,1,2,\ldots$$

Exponential Distribution

FS: Distributions
  • PDF: $f(t) = \lambda e^{-\lambda t}$, CDF: $F(t) = 1 - e^{-\lambda t}$
  • Mean: $E[T] = 1/\lambda$, Variance: $\sigma^2 = 1/\lambda^2$
  • Memoryless property: $P[T > t+s \mid T > s] = P[T > t]$

Merging & Splitting

  • Merging: Independent PP($\lambda_1$) + PP($\lambda_2$) = PP($\lambda_1 + \lambda_2$)
  • Splitting: PP($\lambda$) $\to$ PP($\lambda p_i$) with probability $p_i$
  • Min of exponentials: $\min(\text{Exp}(\lambda_1), \text{Exp}(\lambda_2)) \sim \text{Exp}(\lambda_1 + \lambda_2)$
  • PASTA: Poisson Arrivals See Time Averages

Hitchhiker's Paradox

FS: Distributions

Mean waiting time $= \frac{1}{2}\!\left(\beta + \frac{\sigma^2}{\beta}\right)$ where $\beta$ = mean interarrival, $\sigma^2$ = variance.

  • Exponential ($\sigma^2 = \beta^2$): wait = $\beta$ (not $\beta/2$!)
  • Deterministic ($\sigma^2 = 0$): wait = $\beta/2$

Common Mistakes

  • Confusing rate $\lambda$ with mean $1/\lambda$
  • Forgetting memoryless property when computing conditional probabilities
  • Not using $\min$ of exponentials for "first to finish" problems

Worked Examples

Exercise 13: Repair Time — Exponential mean $1/3$ hours

What we know: A machine has a repair time that follows an exponential distribution with mean $1/3$ hours (i.e., 20 minutes on average). FS: Distributions

Step 1: Identify the rate parameter.
The exponential distribution has mean $= 1/\lambda$. We're told mean $= 1/3$ hour, so: $$\lambda = \frac{1}{\text{mean}} = \frac{1}{1/3} = 3 \text{ per hour}$$

Step 2: Recall the exponential CDF.
For an exponential distribution, the probability of taking longer than time $t$ is: $$P(T > t) = e^{-\lambda t}$$ This comes from the survival function: $1 - F(t) = 1 - (1 - e^{-\lambda t}) = e^{-\lambda t}$.

(a) $P(T > 1/2)$ — probability repair takes more than 30 minutes.
Here $t = 1/2$ hour (30 minutes). Plug into the formula: $$P(T > 1/2) = e^{-3 \times 1/2} = e^{-3/2} = e^{-1.5} \approx 0.2231$$

(b) $P(T \geq 10.5 \mid T > 10)$ — given the machine has been in repair for 10 hours, what is the probability it takes at least another 30 minutes?
This is where the memoryless property kicks in. For exponential distributions: $$P(T > s + t \mid T > s) = P(T > t)$$ In words: "given you've already waited $s$ hours, the probability of waiting at least $t$ more hours is the same as if you just started." The exponential distribution has no memory of how long it's been going.

Here $s = 10$ hours and $t = 0.5$ hours: $$P(T \geq 10.5 \mid T > 10) = P(T \geq 0.5) = e^{-3 \times 0.5} = e^{-1.5} \approx 0.2231$$

Conclusion: Both answers are $e^{-3/2} \approx 0.2231$. The memoryless property means the machine does not "remember" it has been running for 10 hours — the remaining time always has the same distribution.
Exercise 14: Store with 4 customers, service rate $\mu$

What we know: You walk into a store with a single server. There are 4 customers ahead of you (1 being served, 3 in the queue). Each service time is $\text{Exp}(\mu)$. What is your expected total time until you leave?

Step 1: Identify who you're waiting for.
You need to wait for:

  • The person currently being served to finish
  • The 3 people in the queue ahead of you to each be served
  • Your own service
That's $1 + 3 + 1 = 5$ service times in total.

Step 2: Apply the memoryless property to the current customer.
The person currently being served has already been there for some unknown time. Normally, you'd think their remaining time should be less than a full service. But the exponential is memoryless: $$P(T > s + t \mid T > s) = P(T > t)$$ So the remaining service time of the current customer is still $\text{Exp}(\mu)$ — exactly the same distribution as a fresh service. The past is forgotten.

Step 3: Add up the expected times.
Each of the 5 service times is $\text{Exp}(\mu)$ with mean $1/\mu$. Since expected values add up: $$E[\text{total time}] = 5 \times \frac{1}{\mu} = \frac{5}{\mu}$$

Conclusion: Expected total time $= \frac{5}{\mu}$. The key insight is that thanks to the memoryless property, the current customer's remaining service time is the same as a full service time.
Exercise 15: Two cash registers — Aline, Bert, Caroline

What we know: A shop has 2 cash registers.

  • Register 1: Aline is being served (no one behind her).
  • Register 2: Bert is being served, and Caroline is waiting behind him.
Caroline arrives last. We want: $P(\text{Caroline leaves before Aline})$, i.e., both Bert and Caroline finish before Aline.

Scenario 1: Deterministic service (exactly 5 min each).

Step 1: Aline finishes at $t = 5$ min.
Step 2: Bert finishes at $t = 5$ min, then Caroline starts. Caroline finishes at $t = 5 + 5 = 10$ min.
Step 3: Caroline finishes at $t=10$, Aline at $t=5$. Aline always leaves first.
$$P(\text{Caroline before Aline}) = 0$$

Scenario 2: Discrete service times $\{1, 2, 3\}$ each with probability $1/3$.

Step 1: For Caroline to leave before Aline, we need Bert's time + Caroline's time < Aline's time, i.e., $B + C < A$.
Step 2: There are $3^3 = 27$ equally likely $(B, C, A)$ triples. We need $B + C < A$.
The smallest possible $B + C = 1 + 1 = 2$, and the largest $A = 3$. So $B + C < A$ is possible only when $B = 1, C = 1, A = 3$.
(All other combinations have $B + C \geq A$: e.g. $B=1, C=2 \Rightarrow B+C=3$, which is not strictly less than $A=3$.)
$$P(\text{Caroline before Aline}) = \frac{1}{27}$$

Scenario 3: Exponential service $\text{Exp}(\mu)$.

Step 1: Set up the problem.
Let $A \sim \text{Exp}(\mu)$ (Aline), $B \sim \text{Exp}(\mu)$ (Bert), $C \sim \text{Exp}(\mu)$ (Caroline), all independent.
Caroline leaves at time $B + C$. We need $P(B + C < A)$.

Step 2: Use the competing exponentials approach.
While Bert and Aline are both being served, they are two competing exponentials. The key question is: who finishes first? $$P(\text{Bert finishes first}) = \frac{\mu}{\mu + \mu} = \frac{1}{2}$$ If Aline finishes first (probability $1/2$), Caroline can never leave before Aline — game over.

Step 3: If Bert finishes first (probability $1/2$).
Caroline starts service. Now Caroline and Aline are competing. By the memoryless property, Aline's remaining time is still $\text{Exp}(\mu)$, and Caroline's fresh service is $\text{Exp}(\mu)$. $$P(\text{Caroline finishes before Aline} \mid \text{Bert finished first}) = \frac{\mu}{\mu + \mu} = \frac{1}{2}$$

Step 4: Combine.
$$P(\text{Caroline before Aline}) = P(\text{Bert first}) \times P(\text{Caroline first} \mid \text{Bert first})$$ $$= \frac{1}{2} \times \frac{1}{2} = \frac{1}{4}$$

Conclusion: $P(\text{Caroline before Aline}) = 1/4$ for exponential service. The memoryless property is crucial: after Bert finishes, Aline's remaining time resets, giving Caroline a fair 50-50 chance.
Exercise 16: Two Light Bulbs — $P(\text{bulb 1 fails first})$

Problem: Bulb 1 is burning from time $t=0$ with lifetime $\sim \text{Exp}(\lambda_1 = 0.02)$. Bulb 2 is switched on at $t = 8$ with lifetime $\sim \text{Exp}(\text{mean} = 30)$, i.e. $\lambda_2 = 1/30$. What is $P(\text{bulb 1 is the first to fail})$? FS: Distributions

Step 1: Identify the two scenarios.
Bulb 1 can fail first in two ways:

  • Case A: Bulb 1 fails before $t = 8$ (before bulb 2 is even switched on).
  • Case B: Both bulbs survive to $t = 8$, and then bulb 1 fails before bulb 2.

Step 2: Compute Case A.
$P(T_1 < 8) = 1 - e^{-0.02 \times 8} = 1 - e^{-0.16} = 0.14786$

Step 3: Compute Case B using the memoryless property.
$P(T_1 \geq 8) = e^{-0.16} = 0.85214$.
By the memoryless property of the exponential, the remaining lifetime of bulb 1 after $t = 8$ is still $\text{Exp}(0.02)$, independent of having survived 8 hours.
For two competing exponentials, $P(\text{Exp}(\lambda_1) < \text{Exp}(\lambda_2)) = \frac{\lambda_1}{\lambda_1 + \lambda_2}$:
$$P(\text{bulb 1 fails first} \mid \text{both survive to } t{=}8) = \frac{0.02}{0.02 + \frac{1}{30}} = \frac{0.02}{0.02 + 0.0\overline{3}} = \frac{0.02}{0.05\overline{3}} = 0.375$$

Step 4: Combine.
$$P(\text{bulb 1 first}) = P(T_1 < 8) + P(T_1 \geq 8) \cdot \frac{\lambda_1}{\lambda_1 + \lambda_2}$$ $$= 0.14786 + 0.85214 \times 0.375 = 0.14786 + 0.31955 = 0.4674$$

Conclusion: $P(\text{bulb 1 fails first}) = 0.4674$.
Exercise 17: Turtle Crossing — Poisson rate 10/hr

What we know: Cars pass a road according to a Poisson process with rate $\lambda = 10$ per hour. A turtle needs a gap of $m$ minutes with no cars to safely cross. What is the probability it can cross?

Step 1: From the formula sheet to a Poisson process. FS: Distributions
We need: "no cars arrive in an interval of length $m$ minutes."

The formula sheet gives the Poisson PMF: $$p(x) = \frac{e^{-\lambda}\,\lambda^x}{x!}$$ Here $\lambda$ is the expected number of events. For a Poisson process with rate $\lambda_{\text{rate}}$ events per unit time, the expected count over a time window $t$ is $\lambda_{\text{rate}} \cdot t$. Substitute $\lambda = \lambda_{\text{rate}} \cdot t$: $$P[N(t) = k] = \frac{(\lambda_{\text{rate}}\, t)^k}{k!}\, e^{-\lambda_{\text{rate}}\, t}$$ We want $k = 0$ (zero cars).

Step 2: Convert units and set up.
The rate $\lambda_{\text{rate}} = 10$ is in per hour, but $m$ is in minutes. Convert: $t = m/60$ hours.

Step 3: Plug into the Poisson formula with $k = 0$.
$$P[N(m/60) = 0] = \frac{(10 \times m/60)^0}{0!} \cdot e^{-10 \times m/60}$$ Since anything to the power $0$ is $1$, and $0! = 1$: $$= 1 \times e^{-10m/60} = e^{-m/6}$$

Step 4: Calculate for specific values.
For $m = 5$ minutes: $$P = e^{-5/6} = e^{-0.8333} \approx 0.4346$$ About a 43% chance — not bad for the turtle!

For $m = 20$ minutes: $$P = e^{-20/6} = e^{-3.333} \approx 0.0357$$ Only about 3.6% — a 20-minute gap is very unlikely with 10 cars per hour.

Conclusion: $P(\text{safe crossing}) = e^{-m/6}$. For 5 min: $\approx 43.5\%$. For 20 min: $\approx 3.6\%$. The key idea is that "zero events in a time window" uses the Poisson PMF with $k = 0$.
Exercise 18: Sweater Orders — Merging and Splitting

What we know: An e-business sells sweaters online in three colours: white, blue, and red. Orders for each colour arrive as independent Poisson processes. FS: Distributions

  • White sweaters: $\lambda_W = 10$ per hour
  • After a white sweater is ordered, it takes on average 2 minutes until a red sweater is ordered
  • On average, 110 orders for blue or red sweaters are placed per hour
Each order takes on average 4 minutes to pack with a special packing device. Questions:
  1. What is the arrival rate for red sweaters?
  2. What is the mean interarrival time for blue sweaters?
  3. The utilization of the packing devices is 80%. How many packing devices are there?

Step 1: Find the red sweater rate (memoryless property).
Red orders follow a Poisson process, so inter-arrival times are exponential. By the memoryless property, the time from any moment until the next red order has the same exponential distribution — it doesn't matter that a white order just happened.
Average wait = 2 minutes $= 2/60 = 1/30$ hour, and for an exponential distribution $E[T] = 1/\lambda$: $$\lambda_R = \frac{1}{1/30} = 30 \text{ per hour}$$

Step 2: Find the blue sweater rate and its mean interarrival time.
Blue + red together arrive at 110/hr. Since red alone is 30/hr: $$\lambda_B = 110 - \lambda_R = 110 - 30 = 80 \text{ per hour}$$ The mean interarrival time for blue sweaters is: $$E[T_B] = \frac{1}{\lambda_B} = \frac{1}{80} \text{ hours} = \frac{1}{80} \times 60 = 0.75 \text{ minutes} = 45 \text{ seconds}$$

Step 3: Find the total arrival rate (merging).
When you merge independent Poisson processes, you add the rates: $$\lambda_{\text{total}} = \lambda_W + \lambda_R + \lambda_B = 10 + 30 + 80 = 120 \text{ per hour}$$

Step 4: Number of packing devices.
Utilization is not on the formula sheet — from lectures: $\rho = \lambda \cdot E[S] / N$, where $\rho$ is the fraction of time a device is busy, $E[S]$ is the mean service time, and $N$ is the number of devices.
Each order takes 4 min $= 4/60 = 1/15$ hour to pack: $$\rho = \frac{\lambda_{\text{total}} \cdot E[S]}{N} = \frac{120 \times 1/15}{N} = \frac{8}{N}$$ We need $\rho = 0.8$: $$\frac{8}{N} = 0.8 \implies N = \frac{8}{0.8} = 10$$

Conclusion: (1) $\lambda_R = 30$ per hour (memoryless property). (2) Mean interarrival time for blue sweaters = 45 seconds. (3) $N = 10$ packing devices.
Exercise 19: Fitness Center — Poisson Superposition

Problem: Men arrive at a fitness center as a Poisson process with rate 30 per hour. Women arrive independently as a Poisson process. FS: Distributions

  1. When a man arrives, the average time until the next woman is 3 minutes. What is the arrival rate of women?
  2. When a woman arrives, what is the average waiting time for the next person (man or woman)?
  3. Each person spends on average 30 minutes. The utilization of the facilities is 0.8367. How many facilities are there?

Step 1: Part (1) — Women's arrival rate.
If women arrive as a Poisson process with rate $\lambda_W$, the inter-arrival time is $\text{Exp}(\lambda_W)$ with mean $1/\lambda_W$. By the memoryless property, the time until the next woman (from any point) has the same distribution.
Average 3 minutes $= 3/60 = 0.05$ hours, so $\lambda_W = 1/0.05 = 20$ per hour.

Step 2: Part (2) — Average wait for next person.
The merged process (men + women) is Poisson with rate $\lambda_M + \lambda_W = 30 + 20 = 50$ per hour.
Mean inter-arrival time $= 1/50$ hour $= 60/50 = 1.2$ minutes $= 72$ seconds.

Step 3: Part (3) — Number of facilities.
Total demand $= 50$ people/hour $\times$ 0.5 hours/person $= 25$ person-hours per hour.
With utilization $\rho = 0.8367$: each facility provides $\rho = 0.8367$ person-hours per hour.
Number of facilities: $N = 25 / 0.8367 = 29.87 \Rightarrow N = 30$ (round up, since partial facility doesn't exist).

Conclusion: (1) $\lambda_W = 20$ per hour. (2) Average wait = 72 seconds. (3) $N = 30$ facilities.
Exercise 20: Birds — Magpie, Pigeon, Sparrow

What we know: John has a garden fountain visited by pigeons, sparrows, and magpies, arriving as independent Poisson processes.

  • Magpie arrival rate: $\lambda_M = 3$ per hour
  • After a magpie arrives, the average time until a pigeon visits is 4 minutes
  • Total bird arrival rate: 27 birds per hour

Part 1: Find the sparrow arrival rate $\lambda_S$.

Step 1: Derive $\lambda_P$ using the memoryless property.
Pigeons arrive as a Poisson process, so the inter-arrival times are exponential (memoryless). The time until the next pigeon from any moment — including right after a magpie arrives — has the same distribution as a fresh inter-arrival time.
Average time until next pigeon = 4 min = $\frac{1}{15}$ hour, so: $$\lambda_P = \frac{1}{1/15} = 15 \text{ per hour}$$

Step 2: Derive $\lambda_S$.
Total rate = sum of independent Poisson rates: $$\lambda_S = 27 - \lambda_M - \lambda_P = 27 - 3 - 15 = 9 \text{ per hour}$$

Part 1 Answer: $\lambda_P = 15$/hr, $\lambda_S = 9$/hr.

Part 2: A magpie, a pigeon, and a sparrow are all at the fountain. What is the probability the magpie flies away first?

Visit durations are independent exponentials:

  • Pigeon: mean 4 min → rate $\mu_P = 1/4$ per min
  • Sparrow: mean 2 min → rate $\mu_S = 1/2$ per min
  • Magpie: mean 3 min → rate $\mu_M = 1/3$ per min

Step 3: Apply competing exponentials.
For independent exponentials, the probability that one finishes first equals its rate divided by the sum of all rates: $$P(\text{magpie first}) = \frac{\mu_M}{\mu_P + \mu_S + \mu_M} = \frac{1/3}{1/4 + 1/2 + 1/3} = \frac{1/3}{13/12} = \frac{12}{39} = \frac{4}{13} \approx 0.308$$

Part 2 Answer: $P(\text{magpie flies away first}) = 4/13 \approx 0.308$.

Part 3: Pet shop queueing problem.

A pet shop has two registers:

  • Register 1: exponential service, mean 3 min (rate $\mu_1 = 1/3$ per min)
  • Register 2: exponential service, mean 4 min (rate $\mu_2 = 1/4$ per min)
Lisa is being served at register 1, Omar at register 2. Mike and John are waiting in the queue (Mike first, then John). Each person takes the first available register.
Find: $P(\text{Mike is still being served when John finishes})$.

Step 4: First departure — Mike gets a register.
The first register to free up is determined by competing exponentials: $$P(\text{Reg 1 frees first}) = \frac{\mu_1}{\mu_1 + \mu_2} = \frac{1/3}{1/3 + 1/4} = \frac{1/3}{7/12} = \frac{4}{7}$$ $$P(\text{Reg 2 frees first}) = \frac{3}{7}$$ Mike takes whichever register frees first.

Step 5: Second departure — John gets a register.
After Mike starts service, there are two people being served. John waits for the next departure, then takes that register.

Case A (prob $4/7$): Reg 1 freed first → Mike at Reg 1, Omar still at Reg 2.
Now Mike (rate $1/3$) and Omar (rate $1/4$) compete:

  • A1 (prob $4/7$): Mike finishes first → John goes to Reg 1. Mike is done → $P = 0$.
  • A2 (prob $3/7$): Omar finishes first → John goes to Reg 2. Mike at Reg 1 (rate $1/3$), John at Reg 2 (rate $1/4$). $$P(\text{John finishes first}) = \frac{1/4}{1/3 + 1/4} = \frac{3}{7}$$

Case B (prob $3/7$): Reg 2 freed first → Mike at Reg 2, Lisa still at Reg 1.
Now Lisa (rate $1/3$) and Mike (rate $1/4$) compete:

  • B1 (prob $4/7$): Lisa finishes first → John goes to Reg 1. Mike at Reg 2 (rate $1/4$), John at Reg 1 (rate $1/3$). $$P(\text{John finishes first}) = \frac{1/3}{1/4 + 1/3} = \frac{4}{7}$$
  • B2 (prob $3/7$): Mike finishes first → John goes to Reg 2. Mike is done → $P = 0$.

Step 6: Combine all cases.
$$P(\text{Mike still serving when John leaves}) = \underbrace{\frac{4}{7} \cdot \frac{3}{7} \cdot \frac{3}{7}}_{\text{Case A2}} + \underbrace{\frac{3}{7} \cdot \frac{4}{7} \cdot \frac{4}{7}}_{\text{Case B1}}$$ $$= \frac{36}{343} + \frac{48}{343} = \frac{84}{343} = \frac{12}{49} \approx 0.245$$

Mock Exam Q2: Poisson Process

Events arrive at rate $\lambda = 6$/hr. Find: (a) $P(\text{no events in 20 min})$, (b) expected time until 3rd event, (c) given no event in first 10 min, probability of no event in next 5 min.

Show Answer

(a) $P(\text{no events in 20 min})$

Step 1: Convert 20 minutes to hours: $t = 20/60 = 1/3$ hour.
Step 2: Use the Poisson PMF with $k = 0$ (zero events): $$P[N(t) = 0] = \frac{(\lambda t)^0}{0!} e^{-\lambda t} = e^{-\lambda t}$$ Step 3: Plug in $\lambda = 6$, $t = 1/3$: $$P = e^{-6 \times 1/3} = e^{-2} \approx 0.1353$$

(b) Expected time until the 3rd event

Step 1: The time until the $n$-th event is the sum of $n$ independent interarrival times: $S_3 = T_1 + T_2 + T_3$.
Step 2: Each $T_i \sim \text{Exp}(6)$ with mean $1/6$ hour.
Step 3: The sum of $n$ independent exponentials is a Gamma (or Erlang) distribution. Its mean is: $$E[S_3] = 3 \times \frac{1}{\lambda} = \frac{3}{6} = 0.5 \text{ hr} = 30 \text{ min}$$

(c) Given no event in first 10 min, $P(\text{no event in next 5 min})$

Step 1: A Poisson process has independent increments: what happens in one time interval is independent of what happened in any non-overlapping interval.
Step 2: So the condition "no events in the first 10 min" gives us no information about the next 5 minutes. The answer is simply: $$P[N(5/60) = 0] = e^{-6 \times 5/60} = e^{-0.5} \approx 0.6065$$ Note: this is not the memoryless property (that's for exponentials) — this is the independent increments property of the Poisson process.

2h Mock Q4: Bank Office Problem

Two tellers with exponential service. Arrivals are Poisson. Apply merging/splitting and memoryless property.

Show Answer

Key concepts to apply:

Step 1: Merging arrivals.
If customers arrive from multiple independent Poisson sources, add the rates to get the total arrival rate: $\lambda = \lambda_1 + \lambda_2 + \cdots$

Step 2: Splitting / routing to tellers.
If a random arrival goes to teller $i$ with probability $p_i$, the arrivals to teller $i$ form a Poisson process with rate $\lambda_i = p_i \lambda$. For example, if the two tellers are equally likely: $\lambda_1 = \lambda_2 = \lambda/2$.

Step 3: Next departure (min of exponentials).
If teller A has service time $\text{Exp}(\mu_A)$ and teller B has service time $\text{Exp}(\mu_B)$, the time until the next departure (whoever finishes first) is: $$\min(\text{Exp}(\mu_A), \text{Exp}(\mu_B)) \sim \text{Exp}(\mu_A + \mu_B)$$ The probability teller A finishes first: $P = \frac{\mu_A}{\mu_A + \mu_B}$.

Step 4: Memoryless property.
If a customer is currently being served and has been for some time, by the memoryless property, their remaining service time has the same exponential distribution as a fresh service. This means you can treat partially-served customers the same as new ones.

Topic 4: Random Variate Generation

Why This Matters

To simulate any system, you need random variables from specific distributions. These algorithms convert $U(0,1)$ into any distribution you need.

Exam Relevance

Inverse transform is the most tested topic. You must integrate PDFs, invert CDFs, and handle piecewise functions. Composition and acceptance-rejection also appear.

Prerequisite Recap

Think of $U(0,1)$ as a "probability dart" thrown at the y-axis of a CDF. The inverse transform "reads off" the corresponding $x$-value. For a piecewise PDF, you integrate piece by piece and invert each segment separately.

Background Knowledge: CDFs, Inverse Functions & Integration
CDFs & Inverse Functions

The CDF $F(x) = P(X \leq x)$ maps values to probabilities in $[0,1]$. If $F$ is strictly increasing, its inverse $F^{-1}(u)$ maps probabilities back to values. Setting $F(x) = u$ and solving for $x$ gives $x = F^{-1}(u)$.

For $\text{Exp}(\lambda)$: $F(x) = 1 - e^{-\lambda x}$. Set $u = 1 - e^{-\lambda x}$, solve: $x = -\frac{1}{\lambda}\ln(1-u)$. This is $F^{-1}(u)$.
Piecewise Integration

When a PDF $f(x)$ is defined in pieces (e.g., different formulas on different intervals), integrate each piece separately. The CDF must be continuous — each piece picks up where the last one left off.

$f(x) = x$ for $0 \leq x \leq 1$, $f(x) = 2-x$ for $1 < x \leq 2$. CDF: $F(x) = x^2/2$ for the first piece, then $F(x) = 1/2 + \int_1^x (2-t)\,dt$ for the second.
Probability as Area Under the PDF

$P(a \leq X \leq b) = \int_a^b f(x)\,dx$ = area under the PDF curve between $a$ and $b$. The total area must equal 1. This geometric view helps verify your integration and catch errors.

For a triangular PDF on $[0, 2]$ with peak at $x=1$: the total area is a triangle with base 2 and height 1, giving area $= \frac{1}{2}(2)(1) = 1$. ✔
Background Knowledge: Acceptance-Rejection Geometry
Geometric Intuition

Acceptance-rejection works by generating points uniformly under the majorising function $t(x)$ and keeping only those that fall under $f(x)$. Think of it as throwing darts at the rectangle (or envelope) defined by $t(x)$: a dart at $(x, y)$ is accepted if $y \leq f(x)$.

The proposal density $r(x) = t(x) / \int t(x)\,dx$ tells us where to throw darts (the x-coordinate), and the ratio $f(x)/t(x)$ tells us the probability of accepting at that location.

For $f(x) = 2x$ on $[0,1]$ with $t(x) = 2$ (constant majoriser): generate $Y \sim U(0,1)$, accept with probability $f(Y)/t(Y) = 2Y/2 = Y$. Near $x = 0$, almost all darts are rejected; near $x = 1$, almost all are accepted.

Visual Guide: How to Apply the Inverse Transform Method

Write the CDF
Integrate the PDF: $F(x) = \int_{-\infty}^{x} f(t)\,dt$. If the PDF is piecewise, integrate each segment and ensure continuity at the boundaries.
Always verify $F(-\infty) = 0$ and $F(\infty) = 1$ before proceeding!
Set $F(x) = U$
Write the equation $F(x) = U$ where $U \sim U(0,1)$. This is the key step — you're "inverting" the CDF.
Solve for $x$
Algebraically isolate $x$ to get $x = F^{-1}(U)$. This gives you the variate generation formula.
$X = F^{-1}(U)$
Handle Piecewise Domains
If the CDF has multiple pieces, determine which piece $U$ falls in by checking the CDF boundaries. For example, if $F(x_1) = p_1$, then $U \leq p_1$ uses the first piece, $U > p_1$ uses the second.
Draw the CDF and mark the boundary probabilities — this makes the case split obvious.
Generate Variates
Draw $U \sim U(0,1)$ and plug into $F^{-1}(U)$ to produce a random variate $X$ with the desired distribution. Repeat for as many variates as needed.
Shortcut: you can use $U$ or $1-U$ interchangeably since both are $U(0,1)$.

Theory

Inverse Transformation Method

FS: Distributions FS: Derivatives Table
  1. Compute CDF: $F(x) = \int_{-\infty}^x f(t)\,dt$
  2. Solve $u = F(x)$ for $x = F^{-1}(u)$
  3. Algorithm: Generate $U \sim U(0,1)$, return $X = F^{-1}(U)$

Acceptance-Rejection Method

  1. Find majorizing function $t(x) \geq f(x)$ for all $x$
  2. Compute $r(x) = t(x)/\!\int t(x)\,dx$ (normalized density)
  3. Algorithm:
    • Generate $Y$ with density $r$
    • Generate $U \sim U(0,1)$ independent
    • If $U \leq f(Y)/t(Y)$, return $X = Y$; else repeat

Composition Method

FS: Derivatives Table
  1. Split $f(x) = p_1 f_1(x) + p_2 f_2(x) + \ldots$ where $\sum p_i = 1$
  2. Normalize each $f_i$ to be a proper PDF
  3. Algorithm:
    • Generate $U_1 \sim U(0,1)$, select component $i$ based on $p_i$
    • Generate $U_2 \sim U(0,1)$, return $X = F_i^{-1}(U_2)$

Common Mistakes

  • Forgetting to normalize sub-densities in composition
  • Not checking that majorizing function actually majorizes everywhere
  • Forgetting continuity at boundaries of piecewise CDFs
  • Mixing up $u$-ranges when inverting piecewise CDFs

Exercise 21: Inverse Transform — Piecewise PDF with $\tfrac{3}{16}x^2$, $\tfrac{5}{8}x^4$, $\tfrac{1}{x^2}$

Problem: Given the piecewise PDF below, find the unknown constant $a$, build the CDF, and derive the inverse transform. FS: Derivatives Table

$$f(x) = \begin{cases} \dfrac{3}{16}x^2 & -2 \leq x < -1 \\[6pt] \dfrac{5}{8}x^4 & -1 \leq x < 1 \\[6pt] \dfrac{1}{x^2} & 1 \leq x \leq a \\[6pt] 0 & \text{otherwise} \end{cases}$$

Step 1: Find $a$ by requiring total probability = 1.

Integrate each piece separately:

$$\text{Piece 1: } \int_{-2}^{-1}\frac{3}{16}x^2\,dx = \frac{3}{16}\cdot\frac{x^3}{3}\bigg|_{-2}^{-1} = \frac{1}{16}\Big[(-1)^3 - (-2)^3\Big] = \frac{1}{16}(-1+8) = \frac{7}{16}$$

$$\text{Piece 2: } \int_{-1}^{1}\frac{5}{8}x^4\,dx = \frac{5}{8}\cdot\frac{x^5}{5}\bigg|_{-1}^{1} = \frac{1}{8}\Big[1 - (-1)\Big] = \frac{2}{8} = \frac{1}{4}$$

$$\text{Piece 3: } \int_{1}^{a}\frac{1}{x^2}\,dx = \left[-\frac{1}{x}\right]_1^a = -\frac{1}{a} + 1 = 1 - \frac{1}{a}$$

Set total = 1:

$$\frac{7}{16} + \frac{1}{4} + 1 - \frac{1}{a} = 1 \quad\Longrightarrow\quad \frac{7}{16} + \frac{4}{16} = \frac{1}{a} \quad\Longrightarrow\quad \frac{11}{16} = \frac{1}{a}$$

$$\boxed{a = \frac{16}{11}}$$

Step 2: Build the CDF $F(x)$ piecewise.

For $-2 \leq x < -1$ (integrate piece 1 from $-2$):

$$F(x) = \int_{-2}^{x}\frac{3}{16}t^2\,dt = \frac{1}{16}\Big[x^3 - (-8)\Big] = \frac{x^3 + 8}{16}$$

Check: $F(-2) = 0$ and $F(-1) = \frac{-1+8}{16} = \frac{7}{16}$ $\checkmark$

For $-1 \leq x < 1$ (add piece 2 starting from $F(-1) = 7/16$):

$$F(x) = \frac{7}{16} + \int_{-1}^{x}\frac{5}{8}t^4\,dt = \frac{7}{16} + \frac{1}{8}\Big[x^5 - (-1)\Big] = \frac{7}{16} + \frac{x^5 + 1}{8}$$

Check: $F(-1) = \frac{7}{16} + 0 = \frac{7}{16}$ $\checkmark$ and $F(1) = \frac{7}{16} + \frac{2}{8} = \frac{7}{16} + \frac{4}{16} = \frac{11}{16}$ $\checkmark$

For $1 \leq x \leq \frac{16}{11}$ (add piece 3 starting from $F(1) = 11/16$):

$$F(x) = \frac{11}{16} + \int_{1}^{x}\frac{1}{t^2}\,dt = \frac{11}{16} + 1 - \frac{1}{x} = \frac{27}{16} - \frac{1}{x}$$

Check: $F(1) = \frac{27}{16} - 1 = \frac{11}{16}$ $\checkmark$ and $F\!\left(\frac{16}{11}\right) = \frac{27}{16} - \frac{11}{16} = 1$ $\checkmark$

Step 3: Determine $U$-intervals for each piece.

From the CDF boundary values:

  • Piece 1: $F$ runs from $0$ to $\frac{7}{16}$  $\Rightarrow$  $U \in \left[0,\;\frac{7}{16}\right)$
  • Piece 2: $F$ runs from $\frac{7}{16}$ to $\frac{11}{16}$  $\Rightarrow$  $U \in \left[\frac{7}{16},\;\frac{11}{16}\right)$
  • Piece 3: $F$ runs from $\frac{11}{16}$ to $1$  $\Rightarrow$  $U \in \left[\frac{11}{16},\;1\right]$

Step 4: Invert each piece to get $X = F^{-1}(U)$.

Piece 1 ($U \in [0, 7/16)$): Set $U = \frac{x^3 + 8}{16}$

$$x^3 = 16U - 8 \quad\Longrightarrow\quad X = (16U - 8)^{1/3}$$

Piece 2 ($U \in [7/16, 11/16)$): Set $U = \frac{7}{16} + \frac{x^5 + 1}{8}$

$$x^5 + 1 = 8U - \frac{7}{2} \quad\Longrightarrow\quad x^5 = 8U - \frac{9}{2} \quad\Longrightarrow\quad X = \left(8U - \frac{9}{2}\right)^{1/5}$$

Piece 3 ($U \in [11/16, 1]$): Set $U = \frac{27}{16} - \frac{1}{x}$

$$\frac{1}{x} = \frac{27}{16} - U \quad\Longrightarrow\quad X = \frac{16}{27 - 16U}$$

Algorithm (Inverse Transform):
  1. Generate $U \sim U(0,1)$.
  2. If $U < \frac{7}{16}$: return $X = (16U - 8)^{1/3}$.
  3. Else if $U < \frac{11}{16}$: return $X = \left(8U - \frac{9}{2}\right)^{1/5}$.
  4. Else: return $X = \dfrac{16}{27 - 16U}$.
Exercise 22: Acceptance-Rejection for $f(x) = \tfrac{1}{2}\cos(x)$ on $[-\pi/2, \pi/2]$

Problem: Design an acceptance-rejection algorithm to generate samples from $f(x) = \frac{1}{2}\cos(x)$ on $[-\pi/2, \pi/2]$. FS: Derivatives Table

Step 1: Verify $f(x)$ is a valid PDF.
$$\int_{-\pi/2}^{\pi/2} \frac{1}{2}\cos(x)\, dx = \frac{1}{2}[\sin(x)]_{-\pi/2}^{\pi/2} = \frac{1}{2}(1 - (-1)) = 1 \checkmark$$

Step 2: Choose a majorising function $t(x)$.
We need a function $t(x) \geq f(x)$ for all $x$ in the support, whose shape is easy to sample from. Since $f(x) = \frac{1}{2}\cos(x) \leq \frac{1}{2}$ on $[-\pi/2, \pi/2]$, a simple choice is the constant $t(x) = \frac{1}{2}$.
We could also use $t(x) = 1$ (valid since $1 \geq f(x)$ everywhere, but less efficient because it covers more "wasted" area above $f$). Let's use $t(x) = \frac{1}{2}$ for better efficiency.

Step 3: Derive the proposal density $r(x)$.
The proposal density is proportional to $t(x)$: $$r(x) = \frac{t(x)}{\int_{-\pi/2}^{\pi/2} t(x)\,dx} = \frac{1/2}{\pi/2} = \frac{1}{\pi} \quad\text{on } [-\pi/2, \pi/2]$$ This is $U(-\pi/2, \pi/2)$. To generate a proposal $Y \sim r$: $Y = \pi(U_1 - \tfrac{1}{2})$ where $U_1 \sim U(0,1)$.

Step 4: Acceptance condition.
Accept $Y$ if $U_2 \leq \frac{f(Y)}{t(Y)}$. Since $t(Y) = \frac{1}{2}$: $$U_2 \leq \frac{f(Y)}{t(Y)} = \frac{\frac{1}{2}\cos(Y)}{\frac{1}{2}} = \cos(Y)$$

Step 5: Write the algorithm.

Algorithm:
  1. Generate $U_1 \sim U(0,1)$.
  2. Set $Y = \pi(U_1 - \frac{1}{2})$ — a sample from $U(-\pi/2, \pi/2)$.
  3. Generate $U_2 \sim U(0,1)$, independent of $U_1$.
  4. If $U_2 \leq \cos(Y)$: return $X = Y$. Otherwise go to step 1.

Step 6: Acceptance probability.
$$P(\text{accept}) = \frac{\int f(x)\,dx}{\int t(x)\,dx} = \frac{1}{\pi/2} = \frac{2}{\pi} \approx 0.637 = 63.7\%$$ On average, $\frac{\pi}{2} \approx 1.57$ proposals are needed per accepted sample.

Conclusion: Using $t(x) = \frac{1}{2}$ gives acceptance condition $U_2 \leq \cos(Y)$ with acceptance probability $2/\pi \approx 63.7\%$. This is more efficient than using $t(x) = 1$ (which would give only $1/\pi \approx 31.8\%$).

  1. Generate $U_1 \sim U(0,1)$. Set $Y = \left(\frac{\pi}{2}+1\right)U_1$ — a sample from $U\!\left(0,\;\frac{\pi}{2}+1\right)$.
  2. Generate $U_2 \sim U(0,1)$, independent of $U_1$.
  3. If $Y \leq 1$ and $U_2 \leq \dfrac{3Y^2}{8}$: return $X = Y$.
  4. Else if $Y > 1$ and $U_2 \leq \cos(Y - 1)$: return $X = Y$.
  5. Otherwise go to step 1.

Step 6: Acceptance probability.

$$P(\text{accept}) = \frac{1}{c} = \frac{9}{4(\pi+2)} \approx \frac{9}{20.566} \approx 0.438 = 43.8\%$$

On average, $c \approx 2.29$ proposals are needed per accepted sample.

Conclusion: The acceptance-rejection algorithm uses a uniform proposal on $[0, \pi/2+1]$ with the piecewise acceptance condition above. The acceptance probability is $\frac{9}{4(\pi+2)} \approx 43.8\%$.
Exercise 25: Inverse Transform + Composition — $f(x)$ with $\tfrac{1}{3}x^3$, $\tfrac{1}{3x}$, $x$ pieces

Problem: Given the piecewise PDF below, find $p$, then generate variates using (a) the inverse transform method and (b) the composition method. FS: Derivatives Table

$$f(x) = \begin{cases} \dfrac{1}{3}x^3 & 0 < x < 1 \\[6pt] \dfrac{1}{3x} & 1 < x < e \\[6pt] x & e \leq x \leq p \\[6pt] 0 & \text{otherwise} \end{cases}$$

Step 1: Find $p$ by requiring total probability = 1.

$$\text{Piece 1: } \int_0^1 \frac{x^3}{3}\,dx = \frac{1}{3}\cdot\frac{x^4}{4}\bigg|_0^1 = \frac{1}{12}$$

$$\text{Piece 2: } \int_1^e \frac{1}{3x}\,dx = \frac{1}{3}\Big[\ln(x)\Big]_1^e = \frac{1}{3}(1 - 0) = \frac{1}{3}$$

$$\text{Piece 3: } \int_e^p x\,dx = \frac{x^2}{2}\bigg|_e^p = \frac{p^2 - e^2}{2}$$

$$\frac{1}{12} + \frac{1}{3} + \frac{p^2 - e^2}{2} = 1 \quad\Longrightarrow\quad \frac{5}{12} + \frac{p^2 - e^2}{2} = 1 \quad\Longrightarrow\quad \frac{p^2 - e^2}{2} = \frac{7}{12}$$

$$\boxed{p = \sqrt{\frac{7}{6} + e^2}}$$

Part (a): Inverse Transform Method

Step 2a: Build the CDF $F(x)$ piecewise.

For $0 < x < 1$:   $F(x) = \dfrac{x^4}{12}$

Check: $F(0) = 0$, $F(1) = \frac{1}{12}$ $\checkmark$

For $1 < x < e$:   $F(x) = \dfrac{1}{12} + \dfrac{\ln(x)}{3}$

Check: $F(1) = \frac{1}{12}$ $\checkmark$, $F(e) = \frac{1}{12} + \frac{1}{3} = \frac{5}{12}$ $\checkmark$

For $e \leq x \leq p$:   $F(x) = \dfrac{5}{12} + \dfrac{x^2 - e^2}{2}$

Check: $F(e) = \frac{5}{12}$ $\checkmark$, $F(p) = \frac{5}{12} + \frac{7}{12} = 1$ $\checkmark$

Step 3a: $U$-intervals and inversions.

  • Piece 1 ($U \in [0, 1/12)$):   $U = \frac{x^4}{12}$  $\Rightarrow$  $X = (12U)^{1/4}$
  • Piece 2 ($U \in [1/12, 5/12)$):   $U = \frac{1}{12} + \frac{\ln x}{3}$  $\Rightarrow$  $\ln x = 3U - \frac{1}{4}$  $\Rightarrow$  $X = e^{3U - 1/4}$
  • Piece 3 ($U \in [5/12, 1]$):   $U = \frac{5}{12} + \frac{x^2 - e^2}{2}$  $\Rightarrow$  $X = \sqrt{2U - \frac{5}{6} + e^2}$
Algorithm (Inverse Transform):
  1. Generate $U \sim U(0,1)$.
  2. If $U < \frac{1}{12}$: return $X = (12U)^{1/4}$.
  3. Else if $U < \frac{5}{12}$: return $X = e^{\,3U - 1/4}$.
  4. Else: return $X = \sqrt{2U - \frac{5}{6} + e^2}$.

Part (b): Composition Method

Step 2b: Identify the weights (the integrals found in Step 1).

$$p_1 = \frac{1}{12}, \qquad p_2 = \frac{1}{3} = \frac{4}{12}, \qquad p_3 = \frac{7}{12}$$

Step 3b: Normalise each sub-density by dividing $f_i^*(x)$ by $p_i$.

$$g_1(x) = \frac{x^3/3}{1/12} = 4x^3 \;\text{ on } (0,1) \qquad g_2(x) = \frac{1/(3x)}{1/3} = \frac{1}{x} \;\text{ on } (1,e) \qquad g_3(x) = \frac{x}{7/12} = \frac{12x}{7} \;\text{ on } [e,p]$$

Step 4b: Find CDF and inverse for each normalised density.

Component 1: $G_1(x) = x^4$ on $(0,1)$  $\Rightarrow$  $X = U^{1/4}$

Component 2: $G_2(x) = \ln(x)$ on $(1,e)$  $\Rightarrow$  $X = e^U$

Component 3: $G_3(x) = \frac{6(x^2 - e^2)}{7}$ on $[e,p]$  $\Rightarrow$  $X = \sqrt{\frac{7U}{6} + e^2}$

Algorithm (Composition):
  1. Generate $U_1 \sim U(0,1)$.
  2. If $U_1 < \frac{1}{12}$: generate $U_2 \sim U(0,1)$, return $X = U_2^{1/4}$.
  3. Else if $U_1 < \frac{5}{12}$: generate $U_2 \sim U(0,1)$, return $X = e^{U_2}$.
  4. Else: generate $U_2 \sim U(0,1)$, return $X = \sqrt{\frac{7U_2}{6} + e^2}$.
Key difference: In inverse transform, a single $U$ determines both which piece and where within it. In composition, $U_1$ selects the component and a fresh $U_2$ generates within it.
Exercise 26: Composition from Vertical Split — $f(x) = (x^2 + \tfrac{1}{2})/c$ on $[-1, \pi/3]$

Problem: Given $f(x) = \frac{x^2 + \frac{1}{2}}{c}$ on $[-1, \pi/3]$, find $c$, split vertically into two sub-densities, and design a composition algorithm. FS: Derivatives Table

Step 1: Find the normalising constant $c$.

$$c = \int_{-1}^{\pi/3}\!\left(x^2 + \tfrac{1}{2}\right)dx = \left[\frac{x^3}{3} + \frac{x}{2}\right]_{-1}^{\pi/3} = \left(\frac{\pi^3}{81} + \frac{\pi}{6}\right) - \left(-\frac{1}{3} - \frac{1}{2}\right) = \frac{\pi^3}{81} + \frac{\pi}{6} + \frac{5}{6} \approx 1.740$$

Step 2: Vertical decomposition.

Split $f(x)$ into its constant and quadratic parts:

$$f(x) = \underbrace{\frac{1}{2c}}_{f_1^*(x)} + \underbrace{\frac{x^2}{c}}_{f_2^*(x)}$$

Step 3: Compute the weights (area under each part).

$$p_1 = \int_{-1}^{\pi/3} \frac{1}{2c}\,dx = \frac{1}{2c}\!\left(\frac{\pi}{3} + 1\right) = \frac{\pi + 3}{6c} \approx \frac{6.142}{10.440} \approx 0.588$$

$$p_2 = 1 - p_1 \approx 0.412$$

(Verify directly: $p_2 = \frac{1}{c}\!\left(\frac{\pi^3}{81} + \frac{1}{3}\right) \approx \frac{0.717}{1.740} \approx 0.412$ $\checkmark$)

Step 4: Normalise each sub-density.

Component 1: $g_1(x) = \frac{f_1^*(x)}{p_1} = \frac{1}{\pi/3 + 1}$ on $[-1, \pi/3]$. This is $U(-1,\;\pi/3)$.

Generate: $X = -1 + \!\left(\frac{\pi}{3} + 1\right)\!U_2$

Component 2: $g_2(x) = \frac{f_2^*(x)}{p_2} = \frac{x^2}{c\,p_2}$. Let $\alpha = c\,p_2 = \frac{\pi^3}{81} + \frac{1}{3}$.

$$g_2(x) = \frac{x^2}{\alpha} \quad\text{on } [-1, \pi/3]$$

Step 5: Derive the inverse CDF for component 2.

$$G_2(x) = \int_{-1}^{x}\frac{t^2}{\alpha}\,dt = \frac{1}{\alpha}\cdot\frac{x^3 + 1}{3} = \frac{x^3 + 1}{3\alpha}$$

Set $G_2(x) = U$ and solve for $x$:

$$x^3 + 1 = 3\alpha U \quad\Longrightarrow\quad x = (3\alpha U - 1)^{1/3}$$

Check: $G_2(-1) = 0$ $\checkmark$,   $G_2(\pi/3) = \frac{\pi^3/27 + 1}{3\alpha} = \frac{\pi^3/27 + 1}{3(\pi^3/81 + 1/3)} = \frac{\pi^3/27 + 1}{\pi^3/27 + 1} = 1$ $\checkmark$

Step 6: Write the composition algorithm.

Algorithm:
  1. Generate $U_1 \sim U(0,1)$.
  2. If $U_1 \leq p_1 \approx 0.588$:
    Generate $U_2 \sim U(0,1)$. Return $X = -1 + \!\left(\frac{\pi}{3} + 1\right)\!U_2$.
  3. Else:
    Generate $U_2 \sim U(0,1)$. Return $X = \!\left(3\alpha\, U_2 - 1\right)^{1/3}$ where $\alpha = \frac{\pi^3}{81} + \frac{1}{3}$.
Conclusion: Composition weights: $p_1 = \frac{\pi+3}{6c} \approx 0.588$ (uniform component), $p_2 \approx 0.412$ (cubic-root component). The vertical split separates the constant and quadratic contributions of the original density.
Exercise 27: Composition — $f(x) = \frac{1}{96}x + \frac{1}{64}x^3$ on $[2,4]$

Topic 5: Simulation Fundamentals

Why This Matters

Discrete-Event Simulation (DES) is the backbone of simulation. Understanding event routines vs timing routines is essential for exam true/false questions and for building any simulation model.

Exam Relevance

True/false conceptual questions, step-through exercises, and Little's Law applications. Know the DES components cold.

Prerequisite Recap

Think of DES like a play: the initialization routine sets the stage and schedules the first scene. The timing routine is the director calling "next scene!" The event routine is the actors performing that scene. The clock only jumps to moments when something happens.

Background Knowledge: Event-Driven Simulation & Queueing
Event-Driven vs Time-Stepped Simulation

Event-driven (DES): the clock jumps from event to event — nothing happens between events, so we skip idle time. Time-stepped: the clock advances by fixed $\Delta t$ increments and we check for changes each step. DES is far more efficient when events are sparse.

A bank queue with 50 customers over 8 hours: DES processes ~100 events (arrivals + departures). Time-stepped at 1-second resolution processes 28,800 steps — mostly empty.
Queueing Notation

In a single-server queue: arrival = customer enters the system, service = customer is being processed, departure = customer leaves. Key state variables: number in queue $L_Q(t)$, number in system $L(t)$, server status (busy/idle).

At $t = 5.2$: server busy, $L_Q = 2$ means 2 customers waiting + 1 being served = $L = 3$ total in system.
State Variables & Statistical Accumulators

State variables track the system's current condition (queue length, server status). Statistical accumulators track cumulative quantities for computing outputs: total waiting time, total busy time, number served, area under $L_Q(t)$.

Average waiting time = $\frac{\sum_{i=1}^{n} W_i}{n}$. Server utilisation = $\frac{\text{total busy time}}{\text{total simulation time}}$. Average queue length = $\frac{\int_0^T L_Q(t)\,dt}{T}$.

Visual Guide: How to Step Through a DES Event Table

Initialise Clock & State
Set simulation clock $t = 0$. Initialise all state variables (queue length = 0, server = idle) and statistical accumulators (total wait = 0, etc.). Schedule the first arrival event.
Pick the Next Event
Scan the future event list (FEL) and select the event with the smallest scheduled time. Advance the clock to that time. This is the "timing routine."
Always process ties consistently (e.g., departures before arrivals at the same time).
Update State Variables
Execute the event routine: if arrival — add to queue or start service; if departure — remove from service, start next customer or set server idle. Update $L_Q(t)$, $L(t)$, server status.
Schedule New Events
An arrival schedules the next arrival and (if server was idle) a departure. A departure may schedule a new departure if the queue is non-empty. Add these to the FEL.
Don't forget to schedule the next arrival when processing an arrival event!
Record Statistics & Repeat
Update accumulators (waiting time, area under $L_Q$, busy time). Return to Step 2 until the stopping condition is met (e.g., $n$ customers served or clock exceeds $T$). Compute final averages.
$\bar{W} = \frac{\sum W_i}{n}, \quad \bar{L}_Q = \frac{\int_0^T L_Q(t)\,dt}{T}, \quad \hat{\rho} = \frac{\text{busy time}}{T}$

Theory

DES Components

FS: Queuing Systems
  • System state: Variables describing the system (queue length, server status)
  • Event list: Scheduled future events, ordered by time
  • Timing routine: Determines next event, advances simulation clock
  • Event routine: Updates state when a specific event executes
  • Initialization routine: Sets initial conditions, places first event on event list

Queueing Notation & Little's Law

FS: Queuing Systems
  • $W_i = D_i + S_i$ (total wait = delay in queue + service time)
  • $\bar{d} = \lim \frac{1}{n}\sum D_i$ (steady-state mean delay)
  • Little's Law: $L = \lambda W$ (mean number in system = arrival rate $\times$ mean time in system)
  • $Q = \lambda D$ (mean queue length = arrival rate $\times$ mean delay)

Common Mistakes

  • "Event routine determines next event" — FALSE. That is the timing routine.
  • "Initialization routine does not place events" — FALSE. It places the first event.
  • "AS-IS simulation is only for comparison" — FALSE. It is needed for validation.
  • Confusing $L$ (number in system) with $Q$ (number in queue).
